The Dispatcher

One layer above the overnight worker. You hand it a task — or it writes its own — and it figures out the type, pulls the right context from memory + repos, picks a specialized agent, makes it plan first, runs it under guardrails, and hands back a clean result. The overnight worker, generalized to any task, any project, any time.

The pipeline
Every task — yours or self-generated — flows through the same 7 stages. Tap a stage.
Two ways in
The task spec
Generalized from the overnight worker's tonight.md + CONTRACT.md. One file, any task. Drop it in .claude/missions/ or fire it from chat/Telegram.







  
Who gets the job

  
The Dispatcher classifies the task, then routes to a specialist — your existing agent teams + Claude Code subagent types. Single agent for simple work; a swarm for broad/parallel work.

  







  
How much rope it gets

  
Autonomy is per-task-type and earned. New task types start at L0; promote toward full-auto only as trust builds. Some things never auto.

  







  
⚠ Blind spots — read before building

  
Two of these can sink it. Surfaced on purpose so they don't ambush you in month three.

  







  
Cheapest viable path

  
Don't build a new system. This is conventions + one dispatcher skill on top of what already runs. Each phase ships standalone value.
